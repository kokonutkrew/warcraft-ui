WoW UI Designer Release Notes

These notes refer to wowuides.exe version 1.1.x, the beta release.

Introduction

This is the WoW UI Designer initially mentioned here. It started off as an idea I had to be able to load in any addon frame and edit it like you would a form in Borland C++ Builder (my development environment of choice until recently). From that it developed into a full IDE, with the ability to edit lua scripts, and xml files in both code and designer views. Hopefully in the months ahead, it will develop into the only choice for WoW interface developers even if the visual designer is not powerful enough for them.

Features

  • Native Win32 application developed in C# 2005 requiring .Net v2.0. Compatible with Windows XP Home(Untested) and Professional(Tested), Windows 2003 Server (Untested), and Windows 2000 Professional (Tested) with the latest service packs;
  • Multiple docking form interface in the style of Microsoft's development environments, provides flexibility for the user and developer;
  • Tight integration with Blizzard's World of Warcraft game, providing editors for their open User Interface standard that use XML and Lua 5.0 standards;
  • Flexible renderer to display any LayoutFrame element defined in an XML file validated against Blizzard's UI.XSD schema;
  • Embedded ActiproSoftware SyntaxEditor provides syntax highlighting for XML and LUA files, and the ability to display calltips in the future;
  • Visual designer to create new forms and edit them. Uses skin files to define what components can be placed, what properties they have, and how to convert between the XML and Windows.Forms;
  • This release WILL NOT handle your existing UI files within the editor, it can only handle files that it itself created.
  • Lots of other stuff yet to do!

Installation

  1. Install all latest service packs: Windows Update.
  2. Install Windows Installer 3.1, if you don't already have it installed. Unfortunately this requires genuine Windows validation, not sure how to get around it.
  3. Install Microsoft .NET Framework Version 2.0 Redistributable Package (x86)
  4. If you have issues later with the Lua interpretor, install the Microsoft .NET Framework Version 1.1 Redistributable Package, not sure if this is required or not.
  5. Ensure that a copy of World of Warcraft is installed correctly on your system. Use regedit.exe to check HKEY_LOCAL_MACHINE/SOFTWARE/Blizzard Entertainment/World of Warcraft/InstallPath to see if you have a good install. You can change the path in the wowuides program options later, but this reduces the problems.
  6. Should now be good to go, run the installer WoW UI Designer v1.1.X.msi and follow the prompts.

Notes

  • As of WoW version 1.8.0, the Blizzard XML files no longer conform to their schema exactly. In order to load these files correctly, there is an option in the General tab of the options dialog to not validate the Blizzard XML files against the schema.
  • The loading process can take a bit of time, but hopefully you wont have to do it as many times as I did when developing it.
  • Many addons do not validate against the Blizzard schema, the program does not attempt to load these files and will display one error about that file. These can be safely ignored unless you want order. You can use Tools | Validate XML to validate the current buffer against the schema, if no errors are displayed then it is valid. You can then use Tools | Reload Addons to attempt to reload all the addons to the file list again.
  • The pixel values for location and size are accurate at 1024x768 resolution, but are scaled down for lower resolutions by the WoW renderer.
  • The image reader looks for .TGA files first, then .BLP files.
  • There are 2 skins installed with the designer with overlapping component types. The Default skin has fewer properties and should be easier for a beginner to start playing with, the Basic skin has most of the properties available exposed and is good for users requiring more control. There is no way to convert a component from one skin to another without recreating it.

Known Bugs

  • Structure view does not work correctly with table element names in the [expr] format if expr is not a straight integer.
  • Lots of potential exceptions are not correctly trapped, debug output not available to users.
  • Application uses a s**t load of memory once everything is loaded (~200MB), will work to reduce this (but it is beta after all).
  • New forms added to a project, but not edited and saved before closing the project will load them back as normal xml files.
  • Font drawing is a bit off, with characters offset a little high and a bit wider than the WoW rendering.
  • Find and Replace dialog is not finished.

To Do

  • Implement Undo throughout, you can use CTRL-Z or the context menu on the edit control to undo its actions, you cannot Undo what you have done in the visual designer.
  • Expand the default skin, and provide an interface to add more skins. Document the skin file format. Maybe rename to Component Libraries?
  • Improve the renderer speed and functionality, so that it more closely represents the WoW experience.
  • Change the property script editor interface so that it is embedded directly into the document view like a normal lua script, thus giving the author access to all the resources possible.
  • Tighten application security to make the current skin files less dangerous.
  • Allow multiple message view windows to be created, each with their own filtering states.

Planned

  • Implement calltips for LUA and XML code. Use information provided on wowwiki to get the current function listing. Eventually provide all the wowwiki information to the developer contextually through help windows.
  • Improve the editor interface so that all the functionality available in ActiproSoftware.SyntaxEditor is exposed to the user, and can be customised fully.
  • Develop the UI section of the WoW sandbox, so that most scripts can be run to see their effect in the renderer.
  • Change the renderer to a DirectX implementation to provide faster drawing speed and more effects on texture drawing than the current GDI+ implementation.
